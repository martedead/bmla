# Contributing to BMLA

Thank you for your interest in BMLA.

## Reporting problems

Please open a GitHub issue and include:

- MATLAB or Python version
- Operating system
- Minimal reproducible example
- Initial point and solver options
- Complete error message or unexpected output

## Proposing changes

Before submitting a substantial change, open an issue describing the motivation and expected behavior.

## Development principles

- Preserve reproducibility
- Add tests for behavior changes
- Document public API changes
- Do not modify frozen research releases in place
